# 共通
- ファイル内のフォーマットになるべく合わせる
- ファイル末尾は空行で終わらせる

# Kotlin(.kt)
コード上やコミットメッセージへ英語以外を残すことは禁止（`markdown`等のドキュメントやチャットへの返信は適切な言語で行うこと）。

フォーマッターに任せる前提で`import`追加時の順序は考えなくてよい。  
また、以下を守る必要がある。

- ワイルドカードインポート禁止
- クラス等をインポートせず参照することは禁止
    - 名前被りが有ればエイリアスを付与すること
- 非推奨な内容は残さず、なるべくリプレースすること
    - 敢えて利用する場合、何故それを利用しても問題が無いかをコメントした上で抑制すること
- 関数の引数数が7以上となった場合は引数をまとめたDTOを導入する
- `.let { x.let { ... } }`のように`it`が複数重なる場合、レシーバーの名前は外側から付ける
- 1関数が30行を超える場合、適切な単位で分割する
- 1ファイルが500行を超える場合、適切な単位で分割する

なお、暗黙の型に注意し、完了前に不要な`as`や`UNCHECKED_CAST`が残されていないか確認すること。

## ライブラリ追加方針
ライブラリを追加する場合、基本はその時点で最新のリリースバージョン（beta版等を除く）を調べた上で指定する。  
なお、指定されているライブラリバージョンを勝手に上書きすることは禁止。

## ファイル命名方針
ユーザーが頻繁に利用する内容に関しては、名前中に`Kur`を含める。  
これは`Kurorm`の略称である。

プラグイン拡張や内部的に利用される内容については、省略せず`Kurorm`と名付ける。
